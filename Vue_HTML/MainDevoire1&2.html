Devoir 1 PHP expert

Génération de contenus et statistiques

Consignes

Les élèves des trois écoles suivantes :

  • - École A ;
  • - École B ;
  • - École C.

Peuvent pratiquer les cinq sports suivants :

  • - boxe ;
  • - judo ;
  • - football ;
  • - natation ;
  • - cyclisme.

Nous devons effectuer les tâches suivantes :

1. Créer la structure de la base de données.

2. Générer aléatoirement à la demande les contenus de la base de données.

3. Restituer la liste des écoles en affichant pour chacune :

  • - le nombre d’élèves ;
  • - le nombre d’élèves pratiquant au moins un sport ;
  • - le nombre d’activités sportives pratiquées ;
  • - la liste des activités sportives pratiquées classées par ordre croissant en fonction du nombre d’élèves qui les pratiquent et en précisant ce nombre pour chacune des activités.

Le résultat de la génération aléatoire fait que le nombre d’élèves par école est variable et que le nombre d’élèves pratiquant un sport est variable. Un élève peut ne pratiquer aucun sport ou bien trois sports maximum.

4. Question complémentaire : Après avoir terminé le développement, quelle évolution fonctionnelle serait-il utile de mettre en œuvre ?

Devoir 2 PHP expert

PHP expert – Application booking

Consignes

L’application doit permettre à un client de réserver une chambre d’hôtel.

Elle est constituée d’une page d’accueil simple, d’un menu de navigation, d’une page listant les hôtels et d’une page de réservation qui présente un formulaire permettant d’effectuer la réservation.

Les données à sauvegarder en base de données sont les suivantes :

  • - client : nom, e-mail ;
  • - hôtel : nom, adresse
  • - chambres : numéro de chambre
  • - booking : date de début de réservation d’une chambre, date de fin de réservation d’une chambre, date de création de la réservation.

Nous travaillerons volontairement avec peu d’informations, afin de privilégier l’architecture et l’élaboration des requêtes.

À vous d’ajouter les éléments nécessaires au bon fonctionnement de l’application.

Ce développement doit être effectué en POO.

Règles de gestion :

  • - Un client qui valide une réservation est enregistré dans la base de données.
  • - Le premier jour de réservation d’un client peut coïncider avec la date de départ d’un autre client.
  • - La première chambre disponible sur la période de réservation souhaitée sera attribuée.
  • - Nous ne gérerons pas la notion d’update des contenus de la base de données.
